# 1. Skill 路由
# ══════════════════════════════════════════════════════════════
# 收到消息后先按意图加载 skill，再做任何事。
#
# 意图 → Skill：
#   搜索/查资料/研究项目    → agent-reach
#   并行/多agent/蜂群/辩论  → claude-agent-plex
#   开发/改代码/写脚本      → dev-partner-v2
#   创建/更新 skill         → skill-development
#   深度研究报告            → deep-research
#
# 联锁规则：
#   加载 agent-reach       → 自动带出 dev-partner-v2
#   加载 claude-agent-plex → 自动带出 dev-partner-v2
#
# WebFetch 禁用场景（用右边工具替代）：
#   GitHub 任何操作         → gh CLI
#   读网页内容              → curl -s "https://r.jina.ai/URL"
#   搜索                    → MCP: searxng / osint-web

# 2. 开发协作流程
# ══════════════════════════════════════════════════════════════
# 收到开发任务 → 先确认需求，缺信息就问，禁止猜着动手
# 完成任务     → 附验证证据（实测截图/命令输出），不准空口说"搞定"
# 大任务       → 先出方案让用户选
# 没要求改     → 不主动改代码

# ══════════════════════════════════════════════════════════════
